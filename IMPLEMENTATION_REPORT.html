AGENT ARCADE · IMPLEMENTATION REPORT

Agent Arcade · redesigned puzzle-first build

This report describes the runnable first build in this repository. Open index.html to use it.

What was built

  • A simplified, puzzle-first interface inspired by the supplied editorial reference: a compact rounded header, a large uncluttered puzzle canvas, and colorful pill tabs.
  • Workspace, Agents, API keys, puzzle selection, and trace inspection are on-demand drawers rather than permanent panels. The puzzle family picker now uses category tabs plus a level selector, so a 100-level collection does not become a 100-row menu.
  • The old marketing-copy column was removed. The puzzle canvas owns the page, with a compact identity/control bar and a live agent console that shows an animated agent, elapsed time, changing phase, and readable milestones while a model is working.
  • Seven Sokoban challenges (including an imported, attributed Microban puzzle and a playable interpretation inspired by the supplied warehouse image) plus three maze challenges. The newly added logic boards were checked against the local solver.
  • Two numbered Klotski boards. Long tiles are explicitly labelled; commands such as 1R are shown in the run strip, and invalid cross-axis movement is rejected.
  • Four jigsaw boards: two 9-piece (3 × 3) and two 16-piece (4 × 4) visual environments. Tiles are selected and placed using a virtual-mouse-style interaction. The Agents drawer can hide or reveal the reference image.
  • Local-only API-key configuration through .env.local, a small Node server, and an API status endpoint that never returns key values.

How the Sokoban agent works

The local demo agent receives the canonical grid state, runs breadth-first search over legal player moves and crate pushes, then visibly applies its found route one action at a time. This makes the browser demo deterministic, free to run, and testable without an API key.

observe board → find shortest legal state path → animate actions
→ record timeline/trace → check all goals → complete

For the no-key demo, “Delegate alternatives” creates visible child-workspace reports. With Claude selected, the local server instead runs a real bounded tool loop: observe board, spawn explorer, write a temporary helper artifact, request a deterministic solver plan, and apply legal moves. The browser shows only short public activity and explorer reports, not private reasoning.

Visual and interaction design

The Sokoban board uses visual crate, goal, wall, and player affordances inspired by Images/Sokoban1.png, without copying its image content. The Klotski board uses a wood, grid, highlighted target, and exit treatment inspired by Images/Klotski1.jpeg and Images/Klotski2.jpeg.

Live provider and jigsaw checks

The local Claude integration was exercised on the advanced three-crate Sokoban: it spawned an Explorer workspace, wrote a private helper note, requested a verified 11-move plan, and solved the visible board. The 16-piece cat jigsaw was tested with the reference supplied (the model returned a region-by-region visual inventory and virtual placements), and again with the reference withheld (the model produced an exploration plan and made zero placements). The cat asset and level provenance are recorded in SOURCES.md.
