Agent Arcade · Technical explainer

What a Free agent actually does

Free agent is the honest, no-route-hints logic-puzzle mode. Claude receives a text representation of the current puzzle and a small set of safe tools. It can reason, delegate to LLM explorers, and write notes in a temporary workspace, but it cannot call the local solver, browse your computer, run shell commands, or access API keys.

The important distinction: the puzzle's ordinary rules are still enforced. “Free” means no strategic route assistance, not that the agent can walk through walls, alter the level, or bypass the game interface.

One run, end to end

1 · BrowserYou choose a level, model, and Free agent.
2 · Local serverIt creates the canonical in-memory puzzle state and holds the API key locally.
3 · ClaudeIt gets a tool contract plus observations; it never receives your key.
4 · Validated eventThe server checks each proposed action against the real puzzle rules.
5 · RenderThe browser animates only committed legal actions and updates the trace.

Step-by-step loop

1

Load a canonical board

app.js loads the selected level from its local puzzle catalogue. For Sokoban, a compact text map defines walls (#), player (@), crates ($), and targets (.). The local server parses the same map into an independent authoritative state: walls, goals, crate positions, player position, and committed actions.

2

Start a Free-mode request

The browser sends the puzzle map, selected model ID, a task, and solverMode: "free" to the local Node server. The server sends a request to Anthropic using the key from its local .env.local. The browser never sees the key.

3

Claude chooses a tool

Claude can inspect the current board, commit a short action batch, ask an LLM explorer for a separate opinion, or save a tiny private note. In Free mode, the server does not include the route-solver tool or the action-checker tool in Claude's available tools.

4

Validate, stream, then render

When Claude calls apply_actions, the server tries each move in its canonical puzzle environment. Only legal moves become streamed action events. The browser receives those events as they happen, applies them to its visible board state, and re-renders the board after each one.

Claude never receives .env.local, API keys, terminal output, or the host filesystem. The Node server uses the key only to make the provider request.

What “sandbox” means here

“Sandbox” is used in two different, deliberately narrow ways in this app. It is not a claim that Claude has a full virtual computer.

Puzzle sandbox

The server has an in-memory copy of the puzzle. apply_actions can affect only that copy. Walls, crates, target conditions, and long-axis Klotski rules are enforced there.

Temporary workspace sandbox

A requested helper file is written under the operating system temporary folder, in a per-run directory. It is not the repository, not the browser, and not your documents folder.

What it is not

It is not a Codex-style full workstation. The model cannot execute the .py note it writes, inspect other local files, start processes, or touch the visible browser outside the puzzle interface.

Why validation remains

Validation prevents malformed or illegal actions from corrupting the game. It does not reveal a plan or make a strategic choice for Free agent.

How the browser stays live

The server does not wait for a complete solution. It responds as newline-delimited JSON (NDJSON), allowing an action to arrive while the agent continues working. The browser queues each event, animates a legal action, updates the board, and then waits for the next event.

startedModel connected and the selected run style is announced.

Why Microban 31 was difficult

Microban 31 is a real multi-crate Sokoban problem. In Free mode, Claude must derive the crate ordering and player routing from the text board itself. It may ask its own LLM explorers for independent ideas, but those are opinions, not a deterministic search result. Sonnet 5 placed the first crates correctly in the test run, then became inefficient at resolving the remaining crate. That is a real limitation of the current Free-agent setup, rather than a hidden solver failure.
