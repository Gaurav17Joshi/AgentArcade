AGENT ARCADE · IMPLEMENTATION REPORT

Agent Arcade · redesigned puzzle-first build

This report describes the runnable first build in this repository. Open index.html to use it.

What was built

  • A simplified, puzzle-first interface inspired by the supplied editorial reference: a compact rounded header, a large uncluttered puzzle canvas, and colorful pill tabs.
  • Workspace, Agents, API keys, puzzle selection, and trace inspection are on-demand drawers rather than permanent panels. The puzzle family picker now uses category tabs plus a level selector, so a 100-level collection does not become a 100-row menu.
  • The old marketing-copy column was removed. The puzzle canvas owns the page, with a compact identity/control bar and a live agent console that shows an animated agent, elapsed time, selected model, accumulating estimated cost, changing phase, and readable milestones while a model is working.
  • Ten Sokoban challenges: the original warm-ups, an image-inspired warehouse, and four attributed Microban boards. The hardest imported board has an 83-move verified local route.
  • Six maze challenges, including three dense 21 × 21 expert boards, plus five numbered 6 × 6 Klotski boards inspired by the supplied images. Long tiles are explicitly labelled; commands such as 2D are shown in the run strip, and cross-axis movement is rejected.
  • Eight jigsaw boards from 3 × 3 through 6 × 6 visual environments. Tiles are selected and placed using a virtual-mouse-style interaction. The Agents drawer can hide or reveal the reference image.
  • Local-only API-key configuration through .env.local, a small Node server, and an API status endpoint that never returns key values.

How the Sokoban agent works

The local demo agent receives the canonical grid state, runs breadth-first search over legal player moves and crate pushes, then visibly applies its found route one action at a time. This makes the browser demo deterministic, free to run, and testable without an API key.

observe board → find shortest legal state path → animate actions
→ record timeline/trace → check all goals → complete

For the no-key demo, “Delegate alternatives” creates visible child-workspace reports. With Claude selected, the local server instead runs a real bounded tool loop for Sokoban, Maze, and Klotski: observe board, optionally spawn an explorer, write a temporary helper artifact, and choose short batches of legal moves. The Agents drawer exposes three explicit live-run styles: Free agent has no route hints; Action checker tests only a batch the agent has proposed itself and never searches; Solver aid exposes a local deterministic completion route only when the agent explicitly requests it. Free agent is the default. Every batch carries a concise public Reason: tied to the visible board. Each action streams to the browser while the agent remains active, so it can inspect the newly visible state and continue. The browser shows only short public activity and explorer reports, not private reasoning.

Visual and interaction design

The Sokoban board uses visual crate, goal, wall, and player affordances inspired by Images/Sokoban1.png, without copying its image content. The Klotski board uses a wood, grid, highlighted target, and exit treatment inspired by Images/Klotski1.jpeg and Images/Klotski2.jpeg.

Live provider and jigsaw checks

The current curated Anthropic preset is claude-sonnet-5; the dated string claude-sonnet-5-20251001 is diagnosed as invalid. Sonnet 5 was tested in-browser on Two Crates: it observed, used the then-available route helper, stated an observable reason, streamed a three-move final batch, and solved in seven verified moves for an estimated $0.0141. Claude Haiku was also tested through the live Maze and Klotski streams: Maze solved in a reasoned 4-action batch and Klotski solved a numbered 6-action route in two batches. A live 5 × 5 cat jigsaw call produced a visual inventory (eyes, ears, chest, foreground), a harness note, and five five-tile placement batches for an estimated $0.0025—instead of a bare tile-number dump. The cat asset and level provenance are recorded in SOURCES.md.
